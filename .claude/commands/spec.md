Interview me in detail about the feature I want to build.
Use the AskUserQuestion tool to ask about:
- Technical implementation details
- UI/UX requirements
- Edge cases and error handling
- Dependencies on existing modules
- Acceptance criteria

Keep interviewing until we've covered everything.
Then write a complete spec to SPEC.md with:
- Overview
- Requirements (numbered)
- Technical approach
- Files to create/modify
- Acceptance criteria (testable)
- Risks and open questions

After the spec is written, tell me to start a fresh session with:
"Run /clear then implement the spec in SPEC.md"
