## 🪙 KONZEPT: FORGE POINTS (FP)

**1 FORGE Point = €0.10**

Alles wird in Punkten abgerechnet. Provisionen, Rewards, Boni.
Auszahlung erst ab Schwelle. Punkte verfallen nach 12 Monaten Inaktivität.

### Warum Punkte statt direkt Geld:
- **Rechtlich sicherer** — Keine direkte Geldauszahlung = weniger Regulierung
- **Retention** — Punkte binden User an die Plattform
- **Gamification** — Levels, Streaks, Achievements treiben Engagement
- **Cash-Flow** — Du zahlst nicht sofort, sondern erst bei Einlösung
- **Flexibilität** — Punkte können für verschiedene Sachen eingelöst werden
- **Steuerlich** — Punkte sind kein Einkommen bis zur Auszahlung

## 🗄️ DATENBANK

```sql
-- Punkte-Konto pro User
CREATE TABLE IF NOT EXISTS forge_points (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) UNIQUE,
  balance INTEGER DEFAULT 0,           -- Aktueller Kontostand
  total_earned INTEGER DEFAULT 0,      -- Insgesamt verdient (für Level)
  total_spent INTEGER DEFAULT 0,       -- Insgesamt ausgegeben
  level INTEGER DEFAULT 1,
  streak_days INTEGER DEFAULT 0,       -- Login Streak
  last_login DATE,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Transaktions-Log (jede Bewegung)
CREATE TABLE IF NOT EXISTS fp_transactions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id),
  amount INTEGER NOT NULL,             -- Positiv = Gutschrift, Negativ = Ausgabe
  type TEXT NOT NULL,                  -- 'commission', 'bonus', 'payout', 'plan_payment', 'shop', 'engagement'
  description TEXT,                    -- "L1 Provision: Anna S. Signal Pro März"
  reference_id UUID,                   -- FK auf affiliate_payouts, trades, etc.
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Achievements
CREATE TABLE IF NOT EXISTS fp_achievements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id),
  achievement TEXT NOT NULL,           -- 'first_blood', 'hot_streak', etc.
  earned_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(user_id, achievement)
);

-- Payout Requests
CREATE TABLE IF NOT EXISTS fp_payouts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id),
  amount_fp INTEGER NOT NULL,          -- Punkte die ausgezahlt werden
  amount_eur DECIMAL NOT NULL,         -- Euro-Wert
  fee_fp INTEGER DEFAULT 0,            -- Gebühr in Punkten
  method TEXT NOT NULL,                -- 'usdt_trc20', 'btc', 'sepa'
  wallet_address TEXT,
  status TEXT DEFAULT 'pending',       -- pending, approved, processing, paid, rejected
  tx_hash TEXT,
  approved_by UUID,
  approved_at TIMESTAMPTZ,
  paid_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_fp_transactions_user ON fp_transactions(user_id, created_at DESC);
CREATE INDEX idx_fp_payouts_status ON fp_payouts(status);
```

## 🔒 SICHERHEITSREGELN

```
Betrugs-Prävention:
  - Max 1 Account pro Person (Email + IP Check)
  - Engagement FP nur bei echtem Login (nicht Bot)
  - Referral FP nur wenn Referral AKTIV zahlt (nicht nur Signup)
  - Review FP nur 1× (nicht 10 Fake-Reviews)
  - Punkte verfallen nach 12 Monaten Inaktivität
  - Minimum 2.000 FP für Auszahlung (verhindert Micro-Fraud)
  
Manipulation Detection:
  - Wenn User sich selbst referiert → Block
  - Wenn 5+ Referrals aus gleicher IP → Manual Review
  - Wenn Referral innerhalb 24h churnt → FP zurückbuchen
  - Wenn Payout-Pattern verdächtig → Admin Alert
```
