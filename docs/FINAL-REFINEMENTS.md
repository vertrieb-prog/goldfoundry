## 🔓 KYC FIX: REIN OHNE HÜRDE, KYC NUR BEI AUSZAHLUNG

### VORHER (falsch):
```
Signup → KYC → zahlen → loslegen ← ZU VIEL REIBUNG
```

### JETZT (richtig):
```
Signup → Email + Passwort → zahlen (Stripe/Crypto) → SOFORT drin
                                                       ↓
                                                  Loslegen, Leute werben,
                                                  Punkte sammeln
                                                       ↓
                                              Irgendwann: Auszahlen wollen
                                                       ↓
                                              JETZT erst KYC:
                                              "Um auszuzahlen, verifiziere
                                               dich einmalig mit Ausweis."
                                                       ↓
                                              Ausweis hochladen → geprüft
                                                       ↓
                                              Auszahlung freigeschaltet ✅
```

### KYC Ablauf bei erster Auszahlung:
```
Partner klickt "Auszahlen" zum ersten Mal:
  ↓
Modal: "Bevor du auszahlen kannst, brauchen wir einmalig 
        deine Verifizierung. Dauert 2 Minuten."
  ↓
Schritt 1: Vollständiger Name + Geburtsdatum + Land
Schritt 2: Ausweis-Foto (Vorderseite + Rückseite)
Schritt 3: Selfie mit Ausweis in der Hand
  ↓
Status: "In Prüfung — normalerweise innerhalb 24h"
  ↓
Admin prüft → Approved / Rejected
  ↓
Approved: Auszahlung wird verarbeitet
           Alle zukünftigen Auszahlungen OHNE erneutes KYC
```

### Warum NUR bei Auszahlung:
- Null Reibung beim Reinkommen (Conversion Rate bleibt hoch)
- Kein Ausweis-Upload schreckt Neulinge nicht ab
- Punkte sammeln motiviert → KYC wird williger gemacht
- Rechtlich sauber (KYC bei Geldtransaktion, nicht bei Produktkauf)
- User hat schon investierte Zeit → bricht nicht ab wegen KYC

```

MONAT 12-24 (Reife):

  Revenue: 5.000 × €39avg = €195.000/Mo
  Provision: ~€68.250/Mo (35%)
  GF Marge: ~€126.750/Mo (65%)

LANGFRISTIG (volle Tiefe):
  Die 50% werden erst erreicht wenn:
  - Viele Diamond/Crown Leader existieren
  - Netzwerke 10+ Level tief sind
  - Matching Bonus voll greift
  - Performance Pool voll ausgeschüttet wird
  
  Das dauert 2-3 JAHRE.
  Bis dahin hast du immer 60-85% Marge.
```

### Warum das GENIAL ist:
```
1. Du versprichst 50% → Leader sind begeistert

3. Die 50% werden erst bei MASSIVEM Wachstum erreicht
4. Und bei massivem Wachstum hast du genug Revenue dass 50% ok ist
5. Die tiefen Level (L10+) bringen nur 0.05-0.1% pro Level
   → Selbst bei riesigen Netzwerken summiert sich das langsam
6. Compression: Inaktive werden übersprungen → weniger Auszahlung
7. Vesting: 3 Monate Lock → Churner kosten dich nichts
8. Builder Packs: Du bekommst €99-699 UPFRONT, zahlst Provision
   erst wenn die Codes eingelöst werden

→ Du hast IMMER positiven Cash Flow.
→ Die 50% sind ein Versprechen das LANGFRISTIG stimmt.
→ Kurzfristig zahlst du deutlich weniger.
```

## 🔧 UPDATED DATENBANK

```sql
-- KYC vereinfacht (nur bei Auszahlung)
CREATE TABLE IF NOT EXISTS partner_kyc (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) UNIQUE,
  status TEXT DEFAULT 'none',         -- none, pending, approved, rejected
  full_name TEXT,
  date_of_birth DATE,
  country TEXT,
  id_front_url TEXT,
  id_back_url TEXT,
  selfie_url TEXT,
  rejection_reason TEXT,
  approved_by UUID,
  approved_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Generational Bonus Log
CREATE TABLE IF NOT EXISTS generational_bonus_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  partner_id UUID REFERENCES profiles(id),
  triggered_by UUID REFERENCES profiles(id),
  old_rank TEXT,
  new_rank TEXT,
  bonus_fp INTEGER,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- First-to-Rank
CREATE TABLE IF NOT EXISTS first_to_rank (
  rank TEXT PRIMARY KEY,
  achieved_by UUID REFERENCES profiles(id),
  bonus_fp INTEGER,
  achieved_at TIMESTAMPTZ DEFAULT now()
);

-- Contests
CREATE TABLE IF NOT EXISTS partner_contests (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_by UUID REFERENCES profiles(id),
  title TEXT NOT NULL,
  type TEXT NOT NULL,               -- 'most_refs', 'best_performance', 'custom'
  prize_fp INTEGER NOT NULL,
  start_date TIMESTAMPTZ,
  end_date TIMESTAMPTZ,
  winner_id UUID,
  status TEXT DEFAULT 'active',
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Anniversary Tracking
ALTER TABLE profiles ADD COLUMN IF NOT EXISTS partner_since TIMESTAMPTZ;
```
