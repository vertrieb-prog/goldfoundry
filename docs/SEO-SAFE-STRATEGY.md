## 🛡️ GOOGLE-SAFE REGELN FÜR JEDE SEITE

### Regel 1: Jede Seite braucht EINZIGARTIGEN DATENWERT

```
SCHLECHT (wird bestraft):
  /kurs/bitcoin-kurs-heute
  "Bitcoin steht bei [PREIS]. Das ist [CHANGE]%."
  → THIN CONTENT. Null Mehrwert. Google ignoriert das.

GUT (wird belohnt):
  /kurs/bitcoin-kurs-heute-2026-03-17
  - Aktueller Preis mit Live-Chart (interaktiv)
  - 24h Range mit Kontext ("Engste Range seit 2 Wochen")
  - Sentiment: "68% der Trader sind Long" (aus echten Daten)
  - Verwandte Assets: "Gold +0.3%, ETH -1.2%" (Korrelation)
  - Nächstes Event: "FOMC in 3 Tagen" (aus Kalender)
  - Intelligence Insight: "BUY in London Session hat 74% WR" 
  - Interaktiver Rechner: "Was wäre wenn du gestern gekauft hättest?"
  - Letzte 5 Trades der Top-Trader (aus Leaderboard)
  - Verwandte Artikel (interne Links, relevant)
  → RICH CONTENT. Einzigartig. Nützlich. Google rankt das.
```

### Regel 2: Template ≠ Duplicate Content

```
SCHLECHT:
  Gleicher Text, nur [ASSET] ausgetauscht.
  /kurs/bitcoin-kurs → "Der Bitcoin Kurs steht bei..."
  /kurs/ethereum-kurs → "Der Ethereum Kurs steht bei..."
  → Google erkennt das als Doorway Pages.

GUT:
  Gleiches LAYOUT, aber komplett unterschiedliche DATEN:
  /kurs/bitcoin-kurs → BTC Chart, BTC Stats, BTC Sentiment,
    BTC-spezifische Korrelationen (Gold, NASDAQ),
    BTC Halving Countdown, BTC On-Chain Metriken
  /kurs/ethereum-kurs → ETH Chart, ETH Stats, ETH Sentiment,
    ETH Gas Fees, ETH Staking Rate, ETH DeFi TVL
  → Gleiche Struktur, KOMPLETT unterschiedlicher Inhalt.
     Das ist wie Zillow: Jede Adresse hat dasselbe Layout
     aber einzigartige Daten. Google liebt das.
```

## ⚡ TEMPLATE ENGINE — Google-Safe Massenseiten

### Wie Template-Seiten NICHT als Thin Content gelten:

```typescript
// JEDE Kurs-Seite ist ein DATEN-PRODUKT, kein Text-Artikel.
// Google behandelt datengetriebene Seiten anders als Content-Seiten.

interface KursSeite {
  // EINZIGARTIGE DATEN (machen die Seite wertvoll)
  asset: string;
  date: string;
  price: number;
  change24h: number;
  high24h: number;
  low24h: number;
  volume: number;
  marketCap: number;           // Crypto only
  
  // INTERAKTIVE ELEMENTE (User Engagement)
  chart: "TradingView embed";  // Echtes interaktives Chart
  calculator: "Profit Rechner"; // Personalisierbar
  
  // KONTEXTUELLE DATEN (einzigartig pro Tag)
  sentiment: { longPct: number; shortPct: number };
  regime: string;              // Aus Intelligence Engine
  nextEvent: string;           // "FOMC in 3 Tagen"
  correlation: Array<{ asset: string; value: number }>;
  
  // COMMUNITY-DATEN (nutzer-generiert)
  topTraders: Array<{ name: string; todayPnL: string }>;
  recentSignals: Array<{ time: string; direction: string; result: string }>;
  
  // AI-GENERIERTER KONTEXT (einzigartig pro Tag, ~100 Tokens)
  dailyInsight: string;        // "Engste Range seit 2 Wochen. FOMC-Warteposition."
  
  // INTERNAL LINKS (kontextuell relevant)
  links: {
    guide: string;             // /trading/bitcoin-guide
    forecast: string;          // /prognose/bitcoin-maerz-2026
    compare: string;           // /vergleich/bitcoin-vs-ethereum
    glossary: string[];        // Begriffe die im Insight vorkommen
    tools: string[];           // Relevante Rechner
  };
}

// RESULTAT: Jede Seite hat ~20 einzigartige Datenpunkte
// + interaktives Chart + personalisierter Rechner
// + Community-Daten + AI Insight.
// Das ist KEIN Thin Content. Das ist ein Daten-Produkt.
```
